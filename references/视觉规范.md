## 六、固定元素 · 右下水印(v2 新增)

每张图右下角自动渲染水印:

```css
:root {
  --watermark: "@饶秋·1023  ·  第 19 期";
}
.slide::after {
  content: var(--watermark);
  position: absolute;
  right: 90px;
  bottom: 50px;
  font-family: "Source Han Serif SC", "Songti SC", serif;
  font-size: 24px;
  color: var(--gold);
  opacity: 0.55;
  letter-spacing: 4px;
}
```

**为什么这样设计**:
- 一句小字代替"全宽品牌栏",省 50-80 px 给主内容
- 9 张图自动一致(改 1 处 CSS 变量 = 全部生效)
- 类似 nexu-io 的"@HTMLAnything · 5/11"克制水印感
- 让粉丝刷到一眼就识别"这是同一系列"

## 七、信息密度规则

**中间小字页**(第 2-6, 8):

每张图必须有完整 4 层结构:
1. **章节标签**(28 px,米金,`── XXX ──`)
2. **主标题**(88 px,1-2 行)
3. **论据 / 例子 / 列表**(38-44 px,多句)
4. **注解 "为什么"**(28-32 px,灰色斜体)

字数:**每张 100-180 字**。
不允许只放金句 + 留白。

## 九、CTA pill 设计(v2 新增 · 第 9 张专用)

第 9 张 CTA 末尾不用段落式文字 + 加粗黑字,改用三个胶囊按钮:

```html
<div class="pills">
  <span class="pill primary">关 注</span>   <!-- 米金底,主 CTA -->
  <span class="pill outline">收 藏</span>   <!-- 米金边,次 CTA -->
  <span class="pill outline">评 论</span>   <!-- 米金边,次 CTA -->
</div>
```

```css
.pill {
  font-size: 40px;
  font-weight: 700;
  padding: 24px 56px;
  border-radius: 100px;
  letter-spacing: 10px;
}
.pill.primary { background: var(--gold); color: var(--ink); border: 2px solid var(--gold); }
.pill.outline { background: transparent; color: var(--gold); border: 2px solid var(--gold); }
```

**合规说明**:
- 关注 / 收藏 / 评论 = 站内三件套,完全合规
- 不再出现"私你 / 加我 / 公众号同名"等违规词
- 视觉指令明确,引导转化率比段落式 CTA 高

## 十一、字数硬约束(A · 借鉴 enhen3/xhs-cards)

> 没有字数上限 = 9 张图全是字的 PPT,读者根本读不完。

### 总字数

**整组 9 张图,中文总字数 ≤ 1000**(英文/标点不计)

超出 = 强制压缩,一句话能讲清楚就不要两句。"少即是多"在小红书是硬规则,不是口号。

### 单页字数上限

| 角色 | 最大字数 | 说明 |
|---|---:|---|
| 01 `cover` 封面 | 主标 ≤ 15 字 + 副标 ≤ 25 字 | 大字钩点击,不是讲故事 |
| 02 `conclusion` 结论 | ≤ 180 字 | 答案 + 1-2 段展开 |
| 03 `mistake` 误区 | ≤ 180 字 | 3 个反例(每个 ≤ 20 字)+ 总结 |
| 04 `insight` 关键判断 | ≤ 180 字 | 三原则,每条标题 ≤ 12 字 + 注解 ≤ 25 字 |
| 05 `prompt` 完整提示词 | ≤ 250 字(prompt 不计) | prompt 主体 + 变量替换说明 |
| 06 `compare` 改前 vs 改后 | ≤ 150 字 | 两栏各 50-60 字 + 注解 |
| 07 `peak` 金句 | ≤ 30 字 | 一句话视觉锤,绝不展开 |
| 08 `scenario` 适用场景 | ≤ 180 字 | 4 条场景,每条标题 ≤ 15 字 + 注解 |
| 09 `cta` CTA | ≤ 80 字 | 一句金句 + 一句理由 + 三个 pill |

### 自检方法

```bash
# 在 example.md 里数中文字数
wc -m example.md  # 粗略,含 markdown 标记
# 或在浏览器里看渲染后的 HTML 文字内容
```

**Rule of thumb**:每张图主文字占画布 ≤ 60% 面积,留白 ≥ 40%。如果挤满了,一定超字数了。

## 十二、不用 emoji + 5 种替代方案(C · 借鉴 zabr1314)

### 为什么不用 emoji

emoji 会破坏饶秋"程前式安静的深度"调性,显得活泼/廉价。Apple Keynote 风格的高级感**完全依赖留白 + 衬线 + 极简符号**。

加 1 个 emoji = 整张图气质垮一档。

### ❌ 禁用清单(常见误用)

- 标题前后:🔥 💡 ⚡ 🎯 ✨ 📌 等"装饰性"emoji
- 列表项首:✅ ❌ 改用 ✓ ✗(下文有用法)
- 章节标签里的所有 emoji

### ✅ 5 种合规替代

| 编号 | 替代方案 | 用在哪 | 实现 |
|---|---|---|---|
| 1 | **纯文字排版** | 大多数场景 | 用字号 / 颜色 / 留白建立层次 |
| 2 | **数字编号圆圈** | 列表 / 步骤 | `<div class="num-badge">i</div>` 米金底白字 |
| 3 | **极简符号**(`·` `—` `▪` `▸` `→`) | 列表 / 分隔 / 引导 | 米金或灰色,字号比正文小 |
| 4 | **首字母圆圈** | 关键概念 | 用衬线首字代替 emoji,米金圆框 |
| 5 | **几何装饰**(线、圆、方) | 装饰 | 只用米金或砖红,小而精致 |

### 例外允许

**只有两种 emoji 是允许的**:
- ✓ 米金色对勾(适用场景 yes)
- ✗ 灰色叉(适用场景 no)

且必须用 `font-family: "Times New Roman", serif`,不要用系统彩色 emoji。

## 十三、AI 插画规范(E · 借鉴 enhen3/xhs-cards · 中长期可选)

> 默认我们不用插画(衬线 + 留白已经够高级)。但如果选题需要"视觉锚"(比如"AI 时代的工作"这种抽象主题),可以加 1-2 张 editorial 插画。

### 何时加插画

- ❌ **第一层(实操)** 不加 — 提示词、SOP 类内容用文字 + 表格就够
- ⚠️ **第二层(判断)** 可选加 1 张 — 在结论页或对比页加 hero 插画做"视觉锚"
- ✅ **第三层(灵魂)** 适合加 1-2 张 — 思想金句、培训反思类,插画提升氛围

### 一组图最多 2 张插画

加多了变成"花哨",失去 Apple Keynote 极简感。

### Style Prefix(所有插画用同一个)

```
Refined editorial illustration with sophisticated muted color palette.
Use ONLY these tones: ink blue #15263D, soft gold #C4A35A, terracotta #B85850,
paper white #FBFBFD, warm gray.
Clean white or warm off-white background. NO text in image.
Subtle textures, depth, magazine-quality composition.
Conceptual and minimal — not photorealistic, not cartoon.
3:4 portrait aspect ratio.
```

### ⛔ BANNED styles(绝对不用)

- 霓虹 / 全息(Neon glow, holographic)
- 深色剪影 / 赛博朋克
- 通用 stock photo / clip-art / 表情包
- 彩虹色 / 渐变饱和度过高
- 任何写实人像 / 写实建筑(跟"安静的深度"冲突)

### 集成方式

在 9-page-master.html 的 02 / 04 / 06 任一页里加 `.mag-img` 槽位:

```html
<div class="mag-img">
  <img src="hero-XXX.png" alt="">
  <div class="caption">— 一句衬线斜体说明(≤ 20 字)</div>
</div>
```

```css
.mag-img {
  margin: 30px 0;
  border-radius: 12px;
  overflow: hidden;
}
.mag-img img { width: 100%; display: block; }
.mag-img .caption {
  font-family: "Source Han Serif SC", serif;
  font-style: italic;
  font-size: 26px;
  color: var(--gray);
  margin-top: 12px;
  text-align: center;
}
```
