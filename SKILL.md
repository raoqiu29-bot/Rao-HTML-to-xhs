# raoqiu-xhs-builder · 饶秋小红书图文构建器

把饶秋老师的内容(公众号文章 / 培训反思 / 思想笔记 / 提示词 SOP)转成**可一键发布的小红书完整包**:
- ✅ 9 张图(自动渲染 1242×1656 PNG)
- ✅ 小红书标题(跟封面闭环、不重复)
- ✅ 小红书正文(钩子句 + 主体 + 评论钩子)
- ✅ 合规标签(8 个左右,已过滤站外引流词)
- ✅ 发布前自检清单(视觉 / 风格 / 合规 / 闭环 12 项)

**Skill 的输出不是"图片",是"可一键发布的完整发布包"**。视觉风格已锁定 Apple Keynote 白底衬线极简——不可改变,只换内容。

## 三条不可破的硬规则

### 规则 1 · 视觉风格锁死(白底 Apple Keynote)

| 项 | 值 |
|---|---|
| 底色 | `#FBFBFD` 白底 |
| 主文字 | `#15263D` 墨蓝 / 衬线 Source Han Serif SC |
| 强调色 | `#B85850` 砖红(关键词)/ `#C4A35A` 米金(装饰线、引号) |
| 主标题字号 | 封面/末页 156-220 px;中间页 88 px |
| 正文字号 | 38-44 px(中间页),不允许全大字 |
| 副文字 | 28-32 px 灰色斜体,作为"为什么"注解 |

### 规则 2 · 9 张节奏(大-小-大,不能全大)

```
01 封面     ★ 大字  (吸引点击)
02 结论     ○ 小字  (把答案讲透)
03 误区     ○ 小字  (具体例子 + 注解)
04 关键判断  ○ 小字  (三原则 + 解释)
05 提示词   ○ 小字  (完整 prompt + 变量替换)
06 改前vs改后 ○ 小字  (左右对比 + 各自"为什么")
07 金句     ★ 大字  (视觉锤)
08 适用场景  ○ 小字  (✓✓✓+ ✗,每条带"为什么")
09 CTA     ★ 大字  (合规版,只关注 + 评论)
```

中间 6 张必须**小字 + 高信息密度**(每张 100-180 字),把一个观点讲透。
全大字 = 9 张滑下来读者啥也没记住。这是饶秋本人 2026-05-08 反馈的硬要求。

### 规则 3 · 合规红线(踩任一限流 / 封号)

绝对不能写:
- ❌ "公众号 / 视频号 / 抖音 / B 站"等其他平台名
- ❌ "全平台同名 / 全网账号"
- ❌ "评论区扣 XX 我私你 / 我发你"(诱导私信)
- ❌ "加我微信 / 加我 V / 看主页加好友"

可以写(站内动作三件套):
- ✅ 关注(下一篇 / 系列继续看)
- ✅ 收藏(图自身可保存,如提示词图)
- ✅ 评论(开放式问题,引粉丝聊卡点)

## 工作流(标准 6 步 · 缺任一步就不算交付)

### Step 1 · 选题归类

读 `~/Desktop/饶秋-自媒体/11-小红书图文规划/01-选题池-30条.md`,确认是哪一层:
- 第一层"能带走"(占 60%)— 提示词卡片 / SOP / 工具实战
- 第二层"你的判断"(占 25%)— AI 第一版 / 工具取舍 / 这个场景别用 AI
- 第三层"你这个人"(占 15%)— 培训反思 / 总监视角 / 思想金句

### Step 2 · 选封面风格

打开 `templates/cover-styles/Apple-Keynote-6款.html`,根据选题选 1 款:

| 编号 | 风格 | 适合 |
|---|---|---|
| A | 长引文式 | 判断型 / 思想型 |
| B | 短金句式 | 金句型 / 灵魂层 |
| C | 反问式 | 痛点共鸣 |
| D | 数字驱动 | 调研 / 数据 |
| E | 答案揭示式(逆向闭环) | 反直觉判断 |
| F | 行动式(大引号) | 第一层实操 / SOP(默认推荐) |

### Step 3 · md 双层设计(v2 推荐方式)

**新流程**(借鉴 nexu-io/html-anything 的 md 双层模式):

```bash
# 复制内容模板(只管内容,不碰 HTML)
cp "~/Desktop/Skills/01-技能-Skills/我的技能-MySkills/raoqiu-xhs-builder/templates/example.md" \
   "~/Desktop/饶秋-自媒体/11-小红书图文规划/<选题>/example.md"
```

用户只改 md 里 9 段内容,每段用 `@section-tag`、`@h-main`、`@body` 等标签填字段。
AI(我)读完 md 自动套到 `9-page-master.html` 模板,出 HTML。

字段标记语法见 `templates/example.md` 末尾"字段标记语法说明"章节。

**旧流程(直接改 HTML,仍可用)**:
```bash
cp "...templates/9-page-master.html" "...<选题>/F风格-完整9张.html"
```
按节奏改 HTML 内容:
- 第 1 张封面 = 大字钩子 + 大引号
- 第 2-6, 8 张 = 小字密集(每张把一个观点讲透)
- 第 7 张大字金句 / 第 9 张大字 CTA(三个 pill:关注/收藏/评论)

**叙事弧线 9 角色**(每张 `.slide` 有 `data-role` 标识):
`cover → conclusion → mistake → insight → prompt → compare → peak → scenario → cta`

### Step 4 · 写"封面 + 标题闭环"文案

封面图说"现象/钩子"(图里的字),小红书发布标题说"价值/承诺"(图下面那行字)。两者**不能重复**。
6 种闭环模式见 `references/封面标题闭环规则.md`。

### Step 6 · 产出"发布包.md"(必做 · 不能跳)

⚠️ **没有发布包 = 这次工作没完成。Skill 的输出不是 9 张图,是"可一键发布的完整包"**。

每篇 9 张图渲染完成后,**必须**在选题目录下创建 `发布包.md`,内容包含:

| 板块 | 内容 |

| ⑤ 自检清单 | 视觉 / 风格 / 合规 / 闭环 4 类共 12 项,逐项打勾 |
| ⑥ 饶秋两个必问 | "这是真实的饶秋吗?""看完能带走什么?" |

## 渲染工具说明(render.js)

依赖 `puppeteer-core` 调用本机 Chrome,不下载额外 Chromium(节省 ~200MB)。
关键技巧:JS 注入移除 `.slide-wrap` 的 522×696 限制 + 移除 `.slide` 的 transform scale,让每张图以原始 1242×1656 渲染后截图。

如果 Chrome 路径不一样(默认 `/Applications/Google Chrome.app/...`),改 `render.js` 头部 `CHROME` 变量。

## 自检清单(发布前必过)

**视觉**
- [ ] 9 张视觉尺寸严格一致(2484×3312)
- [ ] 封面 / 末页大字 ≥ 156 px
- [ ] 中间页正文 38-44 px(不能全大字)
- [ ] 强调色总面积 ≤ 8%

**饶秋文字风格**
- [ ] 没出现"赋能 / 抓手 / 闭环 / 颗粒度 / 链路"
- [ ] 没用"是 X 不是 Y"句式
- [ ] 没用"今天就能带走的一句话"式 CTA

**合规**
- [ ] 全部图 + 配文不出现其他平台名
- [ ] 不出现"私你 / 加我"
- [ ] 引导动作只剩 关注 / 收藏 / 评论

**闭环**
- [ ] 封面文案 ≠ 小红书发布标题
- [ ] 配文有开放式问题(勾评论)
