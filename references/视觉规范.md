## 六、固定元素 · 右下水印(v2 新增)

每张图右下角自动渲染水印:

```css
:root {
  --watermark: "@饶秋·1023  ·  第 19 期";
}
.slide::after {
  content: var(--watermark);
  position: absolute;
  right: 90px;
  bottom: 50px;
  font-family: "Source Han Serif SC", "Songti SC", serif;
  font-size: 24px;
  color: var(--gold);
  opacity: 0.55;
  letter-spacing: 4px;
}
```

**为什么这样设计**:
- 一句小字代替"全宽品牌栏",省 50-80 px 给主内容
- 9 张图自动一致(改 1 处 CSS 变量 = 全部生效)
- 类似 nexu-io 的"@HTMLAnything · 5/11"克制水印感
- 让粉丝刷到一眼就识别"这是同一系列"

## 七、信息密度规则

**中间小字页**(第 2-6, 8):

每张图必须有完整 4 层结构:
1. **章节标签**(28 px,米金,`── XXX ──`)

3. **论据 / 例子 / 列表**(38-44 px,多句)
4. **注解 "为什么"**(28-32 px,灰色斜体)

字数:**每张 100-180 字**。
不允许只放金句 + 留白。

## 九、CTA pill 设计(v2 新增 · 第 9 张专用)

第 9 张 CTA 末尾不用段落式文字 + 加粗黑字,改用三个胶囊按钮:

```html
<div class="pills">
  <span class="pill primary">关 注</span>   <!-- 米金底,主 CTA -->
  <span class="pill outline">收 藏</span>   <!-- 米金边,次 CTA -->
  <span class="pill outline">评 论</span>   <!-- 米金边,次 CTA -->
</div>
```

```css
.pill {
  font-size: 40px;
  font-weight: 700;
  padding: 24px 56px;
  border-radius: 100px;
  letter-spacing: 10px;
}
.pill.primary { background: var(--gold); color: var(--ink); border: 2px solid var(--gold); }
.pill.outline { background: transparent; color: var(--gold); border: 2px solid var(--gold); }
```

**合规说明**:
- 关注 / 收藏 / 评论 = 站内三件套,完全合规
- 不再出现"私你 / 加我 / 公众号同名"等违规词
- 视觉指令明确,引导转化率比段落式 CTA 高

## 十、叙事弧线 · data-role 清单(v2 新增)

每张 `.slide` 上有 `data-role` 属性,标识故事弧线角色:

| 序号 | data-role | 类型 | 字号 | 用途 |
|---|---|---|---|---|
| 01 | `cover` | ★ 大字 | 156 px | 钩子封面 |
| 02 | `conclusion` | ○ 小字 | 88 / 38 | 先把答案给出来 |
| 03 | `mistake` | ○ 小字 | 88 / 38 | 大部分人在做的事(误区) |
| 04 | `insight` | ○ 小字 | 88 / 38 | 关键判断(三原则) |
| 05 | `prompt` | ○ 小字 | 64 / 26 | 完整提示词(等宽块) |
| 06 | `compare` | ○ 小字 | 88 / 26 | 改前 vs 改后双栏 |
| 07 | `peak` | ★ 大字 | 200 px | 金句视觉锤 |
| 08 | `scenario` | ○ 小字 | 80 / 36 | 适用场景(三 ✓ 一 ✗) |
| 09 | `cta` | ★ 大字 | 160 px | 收尾 + pill 按钮 |

**用途**:
- 让 AI / 用户秒懂"这张图该写什么类型内容"

- 借鉴 nexu-io/html-anything 的 `data-title` 命名页设计
