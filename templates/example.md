# 使用说明

这是「内容 / 视觉分离」的双层设计:
- **你只填这一份 md**(只管内容)
- AI 基于这一份 md,自动套到 `9-page-master.html` 模板里出 HTML
- 然后跑 `node render.js` 出 9 张 PNG
- 整个流程你不用碰 HTML

## 02 · Conclusion · 结论(小字 ○)

> 先把答案给出来,带一个引用块强化反常识结论 + 3 段正文展开。

```
@section-tag: 先 说 结 论
@h-main: 真正卡住你的, 是 @em(「角色」没换)。
@quote-mini: 不是 prompt 写得差。是 AI 默认 @em(「无名无姓」), 你没告诉它它是谁。
@body:
  - 你让一个 @b(无名无姓的助理) 写"月度经营汇报", 它写出来必然是一股「赋能 / 抓手 / 闭环」的 @em(AI 味)。
  - 因为它不知道写给谁、用什么口吻、要达成什么。所以输出永远停在 @b("中庸的、平均的、像 AI 的"声音)。
  - 让它先变成一个 @b(具体的人), 再让它写。prompt 不用复杂, 身份必须明确。
@footer-tip: 滑下一张:大部分人在做的事
```

## 05 · Prompt · 完整提示词(小字 ○)

> 米金底等宽字块 + "改 3 个变量套到任何场景"的变量替换提示。

```
@section-tag: 完 整 提 示 词 · 长 按 截 图 保 存
@h-main: 把这段贴进去, 再让 AI 写。
@prompt-box: |
  @key(你是)一个在上市公司担任运营总监的人。

  @key(你的写作特点):
  - 用大白话, 不用「@em(赋能 / 抓手 / 闭环)」
  - 每个观点后面接一个具体数字或具体场景
  - 主动承认不知道的部分, 不装

  @key(这次你要写):月度经营汇报。
  @key(读者是):总经理 + 部门负责人。
  @key(目标是):不是炫成绩, 是把问题摆出来推动决策。

  @key(接下来)我给你这个月的数据和事件,
  请按这个身份写, 先列结构, 我确认后你再展开。
@vars:
  title: 改这 3 个变量, 套到任何场景:
  list:
    - "① @b(\"你是\") → 换成你的身份(运营 / HR / 销售 / 设计…)"
    - "② @b(\"读者是\") → 换成给谁看(领导 / 客户 / 同事…)"
    - "③ @b(\"目标是\") → 换成想达成什么(决策 / 报告 / 复盘…)"
@footer-tip: 下一张:改前 vs 改后
```

## 06 · Compare · 改前 vs 改后(小字 ○)

> 左右双栏对比 + 每栏底部"为什么"注解 + 黑底总结 pill。

```
@section-tag: 同 样 的 素 材
@h-main: 输出差距, @em(一眼就看出)。
@compare:
  bad:
    head: 改前 · 默认 AI
    quote: "本季度业务取得了显著的增长态势, 团队协同效应明显, 在多维度实现了战略目标的高质量落地, 为下一阶段奠定了坚实基础。"
    why: AI 搜罗的是"标准商务文档", 输出永远在"安全的中庸"上停留。没人会读完。
  good:
    head: 改后 · 带身份
    quote: "4 月新单跌了一半多。三个原因, 一个是渠道没盯紧, 一个是定价试错, 一个是我自己交付节奏没把住。"
    why: 身份是具体的运营总监, AI 模仿的是"会承认问题、会担责任的人" — 这才像真的。
@conclusion: 差别不在 AI ── 在你给它的 @em("角色")。
@footer-tip: 下一张:一句话核心
```

# 字段标记语法说明

| 标记 | 含义 | HTML 对应 |
|---|---|---|
| `@em(文字)` | 砖红强调 | `<span class="em">文字</span>` |
| `@b(文字)` | 加粗墨蓝 | `<b>文字</b>` |
| `@key(文字)` | prompt 块里的"你是""读者是"等标签 | `<span class="key">文字</span>` |
| `@section-tag` | 章节标签 (── XXX ──) | `<div class="section-tag">XXX</div>` |
| `@h-main` | 主标题 | `<h2 class="h-main">...</h2>` |
| `@body-text` | 正文段落 | `<p class="body-text">...</p>` |
| `@footer-tip` | 底部翻页提示 | `<div class="footer-tip">XXX ▶</div>` |
| `@quote-mini` | 米金边引用块(结论页用) | `<div class="quote-mini"><div class="q">...</div></div>` |
| `@open-quote` / `@close-quote` | 大引号(封面/金句/CTA 用) | `<div class="open-q">"</div>` |
| `@pills` | CTA 胶囊按钮组 | `<div class="pills"><span class="pill primary">关注</span>...</div>` |

# 工作流(用 md 双层的方式做新选题)

1. **复制本文件** → `~/Desktop/饶秋-自媒体/11-小红书图文规划/<选题>/example.md`
2. **改 frontmatter**:期号、选题、水印
3. **改 9 张图的内容**(只改 @ 标记后面的文字)
4. **召唤 raoqiu-html-to-xhs skill**,说"用这个 md 出 9 张图"
5. AI 读 md → 套到 9-page-master.html → 输出 HTML
6. AI 跑 render.js → 输出 9 张 PNG 到 `导出/`
7. AI 写 `发布包.md`(标题/正文/标签/自检)
8. 你拿到完整发布包,直接发小红书
